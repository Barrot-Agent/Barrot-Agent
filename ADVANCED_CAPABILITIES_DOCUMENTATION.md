## Usage and Activation

### Manual Triggering

```bash
# Trigger specific workflow
gh workflow run Barrot.Reverse.Engineering.Mastery.yml \
  --ref main \
  -f target_domain=mechanical \
  -f analysis_depth=component

gh workflow run Barrot.Comprehensive.Scraper.yml \
  --ref main \
  -f scraper_focus=web3_blockchain \
  -f gap_filling_mode=both

gh workflow run Barrot.AI.Model.Harvester.yml \
  --ref main \
  -f focus_area=specialized_models \
  -f extraction_depth=comprehensive
```

### Scheduled Execution

All workflows run automatically on their defined schedules:
- **Reverse Engineering**: `:00, :30` (every 30 minutes)
- **Comprehensive Scraper**: `:10, :40` (every 30 minutes, staggered)
- **AI Model Harvester**: `:20, :50` (every 30 minutes, staggered)
- **Continuous operation**: Workflows execute indefinitely on schedule

**Note**: Current implementation uses simulated data for demonstration. Production deployment requires actual API connections, web scraping infrastructure, and data processing pipelines as outlined in Future Enhancements section.

### Monitoring

Check workflow status:
```bash
gh run list --workflow=Barrot.Reverse.Engineering.Mastery.yml
gh run list --workflow=Barrot.Comprehensive.Scraper.yml
gh run list --workflow=Barrot.AI.Model.Harvester.yml
```

View logs:
```bash
gh run view <run-id> --log
```
